    The purpose of the project is to analyze the applications of machine learning models in the field
    of Cyber Security. One rising application of ML within this industry is to detect malicious network
    activity. This can come in the form of cyber attacks and intrusions. Training an ML model to
    detect these attacks is essential to improving the efficiency of cybersecurity systems \cite{ids_survey_ci}. We aim to utilize ML to detect anomalies and/or cyber attacks through network traffic.

\section{Problem Statement}
In this project, we study the UNSW-NB15 network intrusion dataset from
Kaggle \cite{kaggle_unsw_nb15}. This dataset represents simulated network traffic as tabular data.
Each row is a network record that is either labelled as normal traffic or a cyber attack, representing a binary 
classification problem. Attacks can also be labelled into multiple categories, such as Fuzzers, Analysis, Backdoors, etc.
Tabular data with labels is well suited for an ML classification problem.

Thus, our task is a binary classification problem: given a network record,
we want to predict whether it corresponds to normal traffic (label 0) or an attack
(label 1). Our main research question is:

\textit{How do different ML models compare on modern tabular network intrusion detection data, and which model is the most robust and scalable in a cyber security setting?}


Using logistic regression as a baseline, we want to see how tree-based models (Random Forest, 
XGBoost) and neural networks compare to our baseline and to each other, and which model provides the 
most robustness in a cyber security setting, balancing both robustness in predicting attacks and scalability in inference speed and large data.

\section{Existing Approaches and Our Direction}


Existing ML approaches to network intrusion detection include fitting tree-based models or
deep learning on a processed dataset. Some use simpler models such as Random Forest
while others use more novel models such as XGBoost, CatBoost, or LightGBM, or even
neural networks depending on the dataset. Most existing work evaluates models using a
fixed decision threshold of 0.5 and reports overall accuracy or F1, without breaking down
performance across attack categories or tuning the threshold to the specific
trade-offs of the intrusion detection setting.

Our method is to compare four models on the same binary classification
task under a shared preprocessing and evaluation pipeline: Logistic
Regression, Multi-Layer Perceptron, Random Forest, and XGBoost. Logistic
Regression serves as the simple linear baseline, while Random Forest provides a
baseline for a standard tree-based model. MLP is used to showcase the power of neural
networks on tabular data and to see if it can outperform tree-based models, while XGBoost
is our advanced method. We chose this model because it extends the course
ideas and our learning about decision trees, random forests, gradient descent, and ensemble models into a
stronger, tree-based ensemble using gradient-boosted trees. Additionally, we wanted to work with
XGBoost, as it is known for being one of the strongest and most scalable models for tabular data,
making it a well-motivated advanced model for our project.
