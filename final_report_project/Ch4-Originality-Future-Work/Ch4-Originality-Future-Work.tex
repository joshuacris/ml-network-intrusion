\chapter{Originality and Future Work}
\begin{spacing}{1.0}

\section{Originality}

Most studies that compare models on network intrusion detection datasets use a default classification threshold of 0.5. For example, More et al.\ \cite{more_2024_unsw} evaluates classifiers on UNSW-NB15 using fixed thresholds and accuracy, without per-category analysis or threshold tuning.
In this project, we adopt a different approach by tuning the classification threshold for each model rather than fixing it. As explained in Section 3, we test values from 0.01 to 0.99 and pick the one that gives the best F1 score, then compare those results to the standard
0.5 threshold.

The reason for this is that in cybersecurity applications, missing an attack (a false negative) is usually far worse than raising a false
alarm. By adjusting the threshold according to this trade-off, we end up with models that are more useful in real-world scenarios rather than benchmarking accuracy.

Instead of only looking at overall metrics, we break down how each model
performs across the nine attack categories in the UNSW-NB15 dataset using the \texttt{attack\_cat} feature. Kasongo and Sun \cite{kasongo_sun_unsw} similarly study UNSW-NB15 but focus on feature selection and overall accuracy without breaking down performance per attack category. Our per-category analysis gives a clearer picture of each model's strengths and weaknesses — for example, Fuzzers was the hardest attack type to detect for all models, while Logistic Regression, despite being the simplest model, actually performed the best on that category.

\section{Future Work}

One possible direction for future work would be to replace the scikit-learn \texttt{MLPClassifier} with a deep learning
framework such as PyTorch or Keras. Unlike scikit-learn’s MLP, these frameworks support dropout, batch normalization, and
per-class loss weighting, enabling custom loss functions such as focal loss that penalise hard-to-classify samples like Fuzzers
more heavily, which directly addresses the weakest category across all models in this study.

Another promising direction is model ensembling, stacking, or experimenting with novel models. In this study, Logistic Regression, Random Forest, XGBoost, and MLP were
each evaluated independently, yet each model showed different strengths across attack categories. For example, Logistic Regression 
achieved the highest Fuzzers recall while tree-based models excelled on other categories. Rather than selecting a single best model,
a stacking approach could combine the predictions of all four models, allowing the ensemble to exploit the 
complementary strengths of each classifier. Experimentation with other novel models such as CatBoost, LightGBM, or other deep learning architecture 
may result in an improvement in performance metrics.

Finally, all models in this study were trained and evaluated exclusively on the UNSW-NB15 dataset, which was collected in a
lab environment in 2015. This raises questions about how well these models would generalize to real-world traffic or more recent datasets
such as CIC-IDS2017. Since network traffic and attack patterns evolve over time, cross-dataset validation would be a valuable next step 
to better assess the true generalizability of these models.
