\section{Dataset Description}

Our dataset, as explained in Section 1.1, is the UNSW-NB15 dataset \cite{kaggle_unsw_nb15}. This dataset contains records representing network traffic labelled as either normal or attack and has 49 columns, such as \texttt{sbytes}, representing transaction bytes from source to destination or \texttt{sloss}, being dropped source packets. The full data dictionary can be found under \texttt{data/raw/data\_dict\_UNSW-NB15\_features.csv}. Our task is binary classfication, to predict whether a record in normal (0) or an attack (1). The dataset is pre-split into a training set of 107,740 records and a test set of 55,945 records after duplicate removal.

\section{Data Exploration}

In our training set, our records are split 48.2\% normal (51,890) and 51.8\% attack (55,850). We have a nearly balanced training set, however, the test set is less balanced, having 61.1\% normal records (34,206) and 38.9\% attack (21,739). In our EDA, we found 14 heavily right-skewed numerical features (such as \texttt{sload, sbytes}) having ranges up to $10^9$. Three categorical features had a large amount of unique values (\texttt{proto, service, state}), as well as 11 pairs with high correlation ($|r| > 0.9$). Figures A.1 - A.3 in the appendix plot the raw numerical data and the log transformations we applied, as well as the categorical frequency plots, and the correlation heatmap.

\section{Data Cleaning and Feature Engineering}

To prevent data leakage, we removed duplicate rows from both training and test sets before preprocessing. We also removed the columns \texttt{id} (row identifier), \texttt{attack\_cat} (the multi-class label), and \texttt{stcpb}/\texttt{dtcpb} (random sequence numbers). All categorical features were one-hot encoded and condensed by putting categories with rare values into an \texttt{other} category. We also applied a log transformation ($log(1 + x)$) on our skewed numerical features to make the data more normal. Numerical features were standardized using \texttt{StandardScaler} for logistic regression and MLP models, while the tree-based models use the raw values as they are invariant to scaling.

\section{Data Representation and Splitting}

Two datasets are derived from our preprocessing pipeline: \textbf{\texttt{(training)/(test)\_lr\_mlp}} (scaled numericals, used by Logistic Regression and MLP) and \textbf{\texttt{(training)/(test)\_xgb\_rf}} (raw numericals, used by Random Forest and XGBoost), which both have 58 columns plus the label. Preprocessing methods such as the \texttt{StandardScaler} are saved with \texttt{joblib}, which are used to process the test set. For all models, hyperparameter tuning uses \texttt{RandomizedSearchCV} with 3-fold cross-validation on the training set, with evaluation on the test set.
